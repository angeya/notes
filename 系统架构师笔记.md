## 系统工程

### 系统工程概念及方法

从多个维度来看待系统工程

1. 是一种组织管理技术。

2. 是为了最好的实现系统的目的，对系统的组成要素、组织结构、信息流、控制机构进行分析研究的科学方法。

3. 从整体出发、从系统观念出发，以求<span style='color:red'>【整体最优】</span>。

4. 利用计算机作为工具，对系统的结构、元素、信息和反馈等进行分析，以达到最优规划、最优设计、最优管理和最优控制的目的。

5. 是一种现代的科学决策方法。

## 信息系统基础

很多信息系统是过时的，考察可能是想要架构师了解信息系统的演化过程。越往后越先进。

| 信息系统的分类                                | 关键点                                                       |
| --------------------------------------------- | ------------------------------------------------------------ |
| 业务处理系统 【TPS】                          | 早期最初级的信息系统【20世纪50-60年代】<br/>功能:数据输入、<font color='red'>数据处理【批处理、OLTP】</font>（管道流架构风格）、数据库维护、文件报表产生 |
| 管理信息系统 【MIS】                          | 高度集成化的人机信息系统<br />金字塔结构:分多个层级          |
| 决策支持系统 【DSS】                          | 由<font color='red'>语言系统</font>、<font color='red'>知识系统</font>和<font color='red'>问题处理</font>系统组成。<br/>用于辅助决策、支持决策。 |
| 专家系统 【ES】                               | 知识+推理=专家系统。人工智能的一个重要分支。                 |
| 办公自动化系统 【OAS】                        | 由计算机设备、办公设备、数据通信及网络设备、软件系统组成。   |
| <font color='red'>企业资源计划 【ERP】</font> | 打通供应链，集成，整合                                       |

### 业务处理系统 (TPS)

【业务处理系统（Transaction Processing System,TPS）】又可称为<font color='red'>电子数据处理系</font>统(Electronic Data Processing System,EDP)，是计算机在管理方面早期应用的最初级形式的信息系统。

<img src="系统架构师笔记.assets/image-20250323164357565.png" alt="image-20250323164357565" style="zoom:67%;" />

TPS是服务于组织关系层次终<font color='red'>最底层、最基础的信息系统</font>。以逐渐退出历史舞台，基本很少单独使用了。

TPS使用了数据流系统架构风格终的批处理架构风格。

### 管理信息系统 (MIS)

【管理信息系统（Manage Information System,MIS）】是由业务处理系统发展而成的，是在TPS基础上<font color='red'>引进大量管理方法对企业整体信息进行处理</font>，并利用信息进行预测、控制、计划、辅助企业全面管理的信息系统。

MIS系统四大部件：<font color='red'>信息源、信息处理器、信息用户和信息管理者</font>。使用了架构风格中的开环和闭环结构。

**开环结构：**本次的结果只能影响下次结果。

<img src="系统架构师笔记.assets/image-20250323165309959.png" alt="image-20250323165309959" style="zoom: 67%;" />

**闭环结构**：不断获取信息和决策，最终导致结果不断受到影响，如空调定温系统。

<img src="系统架构师笔记.assets/image-20250323165522749.png" alt="image-20250323165522749" style="zoom:67%;" />

### 决策支持系统 (DSS)

决策支持系统(Decision Support System,DSS)是一个由<font color='red'>语言系统、知识系统和问题处理系统</font>3个互相关联的部分组成的，基于计算机的系统。

**<font color='red'>DSS应具有的特征</font>：**

- (1)数据和模型是DSS的主要资源。
- (2)DSS用来支援用户作决策而不是代替用户作决策（ES可以代替）。
- (3)DSS<font color='red'>主要用于解决半结构化及非结构化问题</font>。一般是没有最优解问题，需要权衡，结构化问题MIS系统可以处理。
- (4)DSS的作用在于提高决策的有效性（更加正理性）而不是提高决策的效率。

### 专家系统 (ES)

专家系统(Expert System,ES)是一个智能计算机程序系统，其内部含有某个领域具有<font color='red'>专家水平的大量知识与经验</font>，能够利用人类专家的知识和解决问题的方法来处理该领域的问题。属于人工智能的一个分支。

| 系统         | <font color='red'>专家系统</font>                            | 一般计算机系统             |
| ------------ | ------------------------------------------------------------ | -------------------------- |
| 功能         | 解决问题、解释结果、<font color='red'>进行判断与决策</font>  | 解决问题                   |
| 处理能力     | 处理数字与符号                                               | 处理数字                   |
| 处理问题种类 | 多属准结构性或非结构性，<font color='red'>可处理不确定的知识，使用于特定的领域</font> | 多属结构性，处理确定的知识 |

### 电子政务

电子政务主要有3类角色：政府（Government）、企（事）业单位（Business）及公民（Citizen）。如果有第4类就是公务员（Employee）。

系统分类主要按照谁主动发起来划分的。

| 类型 | 应用                                                         |
| ---- | ------------------------------------------------------------ |
| G2G  | 基础信息的采集、处理和利用，如<span style='color:red'>人口信息</span><br />各级政府决策支持 |
| G2E  | 政府内部管理系统                                             |
| G2B  | <span style='color:red'>政府给企业单位颁发</span>**各种营业执照、许可证、合格证、质量认证**等 |
| B2G  | 企业向政府<span style='color:red'>缴税</span>供应各种商品和服务（含竞/投标）、提建议、申诉等 |
| G2C  | 社区公安和水、火、天灾等与公共安全有关的信息、<span style='color:red'>户口、各种证件和牌照的管理</span> |
| C2G  | 个人应向政府缴纳的各种税款和费用、反馈民意<span style='color:red'>（征求群众意见）</span><br />报警服务（盗贼、医疗、急救、火警等） |

### 企业信息化

#### 信息化的概念

信息化是指在国家宏观信息政策指导下，通过<span style='color:red'>信息技术开发</span>、信息产业的发展、信息人才的配置，最大限度地利用信息资源以满足全社会的信息需求，从而加速社会各个领域的共同发展以<span style='color:red'>推进信息社会的过程</span>。

信息化的主体是全体社会成员（<span style='color:red'>政府、企业、团体和个人</span>），时域是一个<span style='color:red'>长期过程</span>，空域是经济和社会的一切领域，手段是<span style='color:red'>先进社会生产工具</span>。

#### 企业信息化目的

企业信息化的具体目标是优化企业业务活动使之更加有效，它的<span style='color:red'>根本目的在于提高企业竞争能力</span>，使得企业具有平稳和有效的运作能力，对紧急情况和机会做出快速反应，为企业内外部用户提供有价值的信息。



#### 涉及三类创新

**【技术创新】**在生产工艺设计、产品设计中使用计算机辅助设计系统，并<span style='color:red'>通过互联网及时了解和掌握创新的技术信息，加快技术向生产的转化</span>。还有，生产技术与信息技术相结合，能够大幅度地提高技术水平和产品的竞争力。

**【管理创新】**按照市场发展的要求，要对企业现有的管理流程重新整合，从作为<span style='color:red'>管理核心的财务、资金管理</span>，<span style='color:red'>转向技术、物资、人力资源管理</span>，并<span style='color:red'>延伸到企业技术创新</span>、工艺设计、产品设计、生产制造过程的管理，进而还要扩展到客户关系管理、供应链的管理乃至发展到电子商务。

**【制度创新】**那些不适应企业信息化的管理体制、管理机制和管理制度必须得到创新。

#### 信息化需求的3个层次

**战略需求**： 目标 <span style='color:red'>【提升组织的竞争能力】</span>

**运作需求**：【实现信息化战略目标】的需要，【运作策略】 的需要，【人才培养】 的需要

**技术需求**：信息技术层面上对 【<span style='color:red'>系统的完善、升级、集成】</span>

> 组织（企业）对信息化的需求是 组织信息化的原动力。3个层次的需求自顶向下。

#### 例题

#### 企业信息化方法

业务流程重构方法：“彻底的、根本性的”<span style='color:red'>重新设计流程</span>。

核心业务应用方法：围绕<span style='color:red'>核心业务</span>推动信息化。

信息系统建设方法：<span style='color:red'>建设信息系统</span>作为企业信息化的重点和关键。

主题数据库方法：建立<span style='color:red'>面向企业的核心业务的数据库</span>，消除“信息孤岛”。

资源管理方法：切入点是为<span style='color:red'>企业资源管理</span>提供强大的能力。如：ERP、SCM。

人力资本投资方法：人力资本理论【注意不是人力资源管理】<span style='color:red'>把一部分企业的优秀员工看作是一种资本</span>，能够取得投资收益。



#### 信息系统战略规划方法

目标：如何通过信息化实现企业战略目标

**第一阶段**
 以数据处理为核心（围绕职能部门需求）

1. <span style='color:red'>关键成功因素法</span>（<span style='color:red'>CSF</span>，Critical Success Factors）：找到<span style='color:red'>关键信息集合、成功因素</span>（抓住主要矛盾），<span style='color:red'>决定开发有限次序</span>

2. <span style='color:red'>战略集合转化法</span>（<span style='color:red'>SST</span>，Strategy Set Transformation）：企业的<span style='color:red'>战略目标、信息集合转换为信息系统的战略目标</span>

3. <span style='color:red'>企业系统规划法</span>（<span style='color:red'>BSP</span>，Business System Planning）：<span style='color:red'>自顶向下的规划</span>和<span style='color:red'>自底向上的实现</span>。(可以使用UC[use/create]矩阵实现子系统划分)

   > 此外还有：投资回收法、征费法、零线预算法、阶石法

**第二阶段**
<span style='color:red'>以企业内部MIS为核心</span>（围绕企业整体需求）

1. 战略数据规划法（SDP，Strategic Data Planning）：<span style='color:red'>主题数据库(整合各个应用数据库)</span>
2. 信息工程法（IE，Information Engineering）
3. 战略栅格法（SG，Strategic Grid）

**第三阶段**
 综合考虑企业内外环境（<span style='color:red'>以集成为核心(如ERP)</span>，围绕企业战略需求）

1. 价值链分析法（VCA，Value Chain Analysis）
2. 战略一致性模型（SAM，Strategic Alignment Model）

### 企业资源计划(ERP)

演化过程：物料需求计划(Material Requirement Planning，MRP) <span style='color:yellow'>==></span> 制造资源计划(MRPII，Manufacturing Resource Planning II)<span style='color:red'>核心是物流，主线是计划</span> <span style='color:yellow'>==></span> 企业资源计划 (ERP，Enterprise Resource Planning) <span style='color:red'>打通了供应链、扩展到了非制造业、重心转移到财务上</span>

### 客户关系管理(CRM)

CRM(Customer Relationship Management)的<span style='color:red'>目的是提高收入</span>。CRM的<span style='color:red'>核心思想就是以客户为中心</span>。

CRM的主要模块：销售自动化、营销自动化、客户服务与支持、商业智能

CRM的价值：提高工作效率，节省开支、提高客户满意度、提高客户的忠诚度

### 供应链管理(SCM)

SCM (Supply Chain Management)理念：强强联合，整合与优化“三流”，打通企业间“信息孤岛”，严格的数据交换标准。

如实现供应商、制造商、分销商和零售商的计划（策略性）、采购、制造、配送、退货等操作。

**SCM信息化的三流**

- <span style='color:red'>信息流（核心）</span> 

  <span style='color:red'>需求信息流（需求方到供应方）</span>：如客户订单、生产计划、采购合同等 

  <span style='color:red'>供应信息流（供应方到需求方）</span>：如入库单、完工报告单、库存记录、可供销售量、提货发运单等

- 资金流（辅助）

- 物流（辅助）

### 商业智能(BI)

商业智能是指利用数据仓库、数据挖掘、在线分析处理（OLAP）等技术，对企业内部和外部的数据进行收集、转换、存储、分析和展示，从大量的数据中提取有价值的信息。从而为企业的管理决策提供支持的一系列过程和工具。

![image-20250326233808091](系统架构师笔记.assets/image-20250326233808091.png)

商业智能和普通应用系统开发的区别

**普通应用系统开发**：应用数据库 <---> OLTP(联机事务处理)。用于支撑业务运作。

<span style='color:red'>**商业智能**</span>：数据仓库 <---> OLAP(联机分析处理)  ---  数据挖掘。主要用于决策分析，分析历史数据预判未来

**数据仓库**

其中数据库和数据仓库的区别如下：

| 数据库                         | 数据仓库【特点】                                             |
| ------------------------------ | ------------------------------------------------------------ |
| 面向应用：按应用组织数据       | <span style='color:red'>面向主题</span>：按主题组织数据      |
| 零散的：一个应用对应一个数据库 | <span style='color:red'>集成的</span>：整个企业对应一个数据仓库 |
| CRUD：增删改查是常态           | <span style='color:red'>相对稳定的（非易失的）</span>：查询为主、基本无修改与删除 |
| 解决当下应用问题               | <span style='color:red'>反映历史变化（时变的）</span>：各个阶段信息都有，并可做预测未来趋势 |

**数据挖掘**

关联分析：挖掘出隐藏在数据间的相互关系。

序列模式分析：侧重点是分析数据间的前后关系（因果关系）。

分类分析：为每一个记录赋予一个标记再按标记分类。

聚类分析：分类分析法的逆过程。

### 数据湖

数据湖是一个存储企业的各种各样原始数据的大型仓库，其中的数据可供存取、处理、分析及传输。

![image-20250327222249261](系统架构师笔记.assets/image-20250327222249261.png)

<span style='color:red'>数据仓库仅支持数据分析处理。数据湖既支持数据分析处理，也支持事务处理。</span>

数据仓库和数据湖的其他区别

| 维度     | 数据仓库                                                     | 数据湖                                                       |
| -------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 数据     | 清洗过的数据结构化的数据                                     | 原始数据结构化，半结构化数据                                 |
| 模式     | 数据存储之前定义数据模式<br />数据集成之前完成大量工作 数据的价值提前明确 | 数据存储之后定义数据模式提供敏捷<br />简单的数据集成数据的价值尚未明确 |
| 存取方法 | 标准SQL接口                                                  | 应用程序，类SQL的程序                                        |
| 优势     | 多数据源集成干净，安全的数据转换一次，多次使用               | 无限扩展性并行执行支持编程框架数据经济                       |

如果只用数据湖，数据提取难度较大，可以对数据湖进行分区，分别存储结构化数据和非结构化数据。或者数据库仓库和数据库一起使用，即湖仓一体化。

### BPR和BPM

面对企业的流程优化一般有两种方式：

BRP【业务流程重组】：<span style='color:red'>颠覆原有流程、彻底性的再设计。</span>
BPM【业务流程管理】：使用PDCA（plan、do、check、action）循环，持续改进。
